# AkasGatha

> **Where ancient sky stories meet modern space science.**

AkasGatha is a generative AI educational web platform that bridges ancient Indian astronomical narratives with verified modern space science. It uses **Groq** (with `openai/gpt-oss-20b` and `openai/gpt-oss-120b`) and a **Retrieval-Augmented Generation (RAG)** system to produce structured, layered answers — keeping mythology and science clearly separated at all times.

---

## Problem Statement

Ancient Indian texts contain rich, documented observations of the sky — planetary movements, eclipses, lunar cycles, nakshatra systems, and cosmic myths. These stories are widely available online, but either stripped of scientific context, or misleadingly presented as literal scientific proof. No accessible platform:

1. Presents ancient sky narratives with genuine cultural respect
2. Explains the **modern science** behind the same phenomena
3. Structurally and visually **separates story from science**
4. Uses AI to make this exploration interactive and curiosity-driven

AkasGatha solves all four.

## AI Integration — Groq

AkasGatha uses **Groq** as its primary AI provider with a two-gate model system:

| Gate | Model | Purpose |
|---|---|---|
| Primary | `openai/gpt-oss-120b` | Main query responses (full structured output) |
| Secondary | `openai/gpt-oss-20b` | Fallback for rate limits or latency spikes |

All AI calls happen **server-side only** via `/api/jigyasa`. No API key is ever sent to the browser.

The system prompt enforces structured JSON output with clearly separated sections (Katha, Vigyan, Satya Setu, Pramaan Matrix). Content safety validation runs on every response before it reaches the user.

---

## RAG and Citation System

AkasGatha uses a **local TF-IDF RAG pipeline** (no external vector database required for development):

- Source corpus: curated classical texts and verified astronomical references
- Chunking: 512 tokens with 50-token overlap
- Retrieval: TF-IDF cosine similarity, top-5 results per query
- Citations: Every AI response includes source references from the retrieval context
- RAG is required for all answers in strict mode (`JIGYASA_REQUIRE_RAG=true`)
- Hybrid mode allows general space/astronomy answers when the corpus is insufficient

---

## Katha and Vigyan Separation

This is a core, non-negotiable design principle of AkasGatha:

- **Katha Mandal** (Story) is always rendered with a distinct visual treatment and labeled "Cultural Story — Not Science"
- **Vigyan Drishti** (Science) is always rendered separately with scientific framing
- The AI prompt contract explicitly forbids mixing the two sections
- Response validation rejects any output that conflates mythology with fact

This structural separation prevents misinformation while respecting cultural heritage.

## Security Features

AkasGatha implements a multi-layer security architecture (Phase 7 hardened):

| Layer | Implementation |
|---|---|
| API key isolation | Groq key never leaves the server; not exposed in any client bundle |
| Input validation | Maximum 2000 characters; structured schema validation |
| Rate limiting | 10 requests per 60-second window per client |
| Content safety | AI response validated against safety schema before delivery |
| Request timeout | 30-second server-side timeout; no hanging connections |
| History limit | Max 8 conversation turns; max 6000 characters total |
| Middleware guard | Next.js middleware validates all `/api/` routes |
| Log sanitisation | Question content logging disabled in production |
| Error sanitisation | Internal errors never leak stack traces or provider details to clients |
| `.env` isolation | `.env.local` is gitignored; `.env.example` contains only empty placeholders |

Full threat model: [`docs/THREAT_MODEL.md`](docs/THREAT_MODEL.md)
Full security guide: [`docs/SECURITY.md`](docs/SECURITY.md)

## Local Setup

### Prerequisites

- Node.js 18+
- npm 9+
- A [Groq API key](https://console.groq.com/keys) (free tier available)

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/aryaaman577/akasgatha.git
cd akasgatha

# 2. Install dependencies
npm ci

# 3. Create your local environment file
cp .env.example .env.local
# Open .env.local and set:
#   AI_PROVIDER=groq
#   GROQ_API_KEY=your_actual_key_here

# 4. Start the development server
npm run dev

# 5. Open in browser
# http://localhost:3000
```

> **Note:** You can use `AI_PROVIDER=mock` to run without a Groq key. The mock provider returns placeholder responses for UI development.

## Known Limitations

- **No streaming in mock mode** — streaming responses require a real Groq API key
- **RAG corpus size** — current corpus covers core Indian astronomical texts; extended topics are being added
- **Cerebras provider disabled** — returns HTTP 402 (payment required); kept in config for future re-enablement
- **3D performance on low-end devices** — React Three Fiber scenes are lazy-loaded but may be slow on older hardware
- **No user authentication** — this is a stateless public tool; no accounts, no personal data stored
