# Report Writing Prompt

> **Purpose**: Complete the internship report with real content, screenshots, and proper formatting.

---

## Prompt

```
You are helping complete the AkasGatha internship report.

FIRST: Read docs/REPORT_DRAFT.md for the report structure.
ALSO READ: brain/PROGRESS_LOG.md for what was actually built and tested.

YOUR TASK:
Finalize the internship report. Fill in placeholders, add technical details, and ensure it demonstrates Gen AI + Cloud Computing skills.

SECTIONS TO COMPLETE:

1. Abstract — Finalize with actual results and metrics
2. Problem Statement — Keep as-is (already well-written)
3. Objectives — Keep as-is, mark which were achieved
4. Proposed Solution — Add implementation details
5. System Architecture — Add actual architecture diagram description
6. Modules — Add actual implementation notes per module
7. Gen AI Usage — Add:
   - Actual prompt engineering approach used
   - Structured output examples (sanitized — no API keys)
   - Content safety measures implemented
   - Example AI response screenshot description
8. Cloud Computing Usage — Add:
   - Docker configuration details
   - AWS EC2 setup details
   - Deployment process
   - Screenshot descriptions
9. Security Measures — Add actual security implementations
10. Testing Summary — Fill in test results (pass/fail)
11. Future Scope — Keep as-is
12. Conclusion — Finalize with actual outcomes
13. References — Verify all links are valid

SCREENSHOT DESCRIPTIONS:
Add descriptions for where screenshots should be inserted:
- [Screenshot: Home page hero section]
- [Screenshot: Topic library grid]
- [Screenshot: AI response with all sections]
- [Screenshot: 3D scene rendering]
- [Screenshot: Docker build output]
- [Screenshot: EC2 instance running]
- [Screenshot: App on public IP]

FORMATTING:
- Professional academic tone
- Clear section numbering
- Tables for structured data
- Code blocks for technical examples (sanitized — no secrets)
- Figure references for screenshots

DO NOT:
- Include actual API keys in the report
- Include actual EC2 IP addresses
- Include personal information beyond name
- Make unverifiable claims
- Present mythology as scientific proof in any example
```
