# Skill: Section Design

> How to design each section of AkasGatha for maximum impact.

---

## Akas Dwar (Celestial Gateway)

| Attribute | Detail |
|---|---|
| **Purpose** | First impression. Draw the user into the cosmic experience. |
| **What it shows** | Title "AkasGatha", tagline, CTA button, atmospheric background |
| **UI style** | Full-viewport hero, dark gradient or starfield background, large cinematic typography, floating particles or subtle animation |
| **What makes it impressive** | Cinematic feel — like opening a space documentary. Smooth entrance animation. Atmospheric depth. |
| **Mistakes to avoid** | Plain white page. No animation. Generic stock image. Cluttered hero. Too much text. |

---

## Akas Granth (Sky Library)

| Attribute | Detail |
|---|---|
| **Purpose** | Browse available topics. Give users starting points for exploration. |
| **What it shows** | Grid of topic cards: planets, nakshatras, eclipses, moon phases, mysteries |
| **UI style** | 3–4 column grid (responsive). Cards with icon/emoji, title, short description. Hover effect with subtle glow. Stagger entrance animation. |
| **What makes it impressive** | Clean grid with rich cards. Each card feels like a portal to a new topic. Hover effects invite clicking. |
| **Mistakes to avoid** | Plain list with no visual interest. Cards all look the same. No hover feedback. Too many topics (keep to 7–10). |

---

## Jigyasa Engine (Curiosity Engine)

| Attribute | Detail |
|---|---|
| **Purpose** | The core interaction — user asks a cosmic question. |
| **What it shows** | Text input, mood selector, topic selector, submit button |
| **UI style** | Centered form. Large input field with placeholder ("What cosmic mystery intrigues you?"). Dropdown or chip selectors for mood and topic. Prominent submit button. |
| **What makes it impressive** | Feels like querying an ancient oracle + modern AI. Input field is inviting. Clear call-to-action. Loading state is cinematic (not just a spinner). |
| **Mistakes to avoid** | Tiny input field. No placeholder text. No loading state. Generic "Submit" button text (use "Explore" or "Seek Answers"). |

---

## Katha Mandal (Story Circle)

| Attribute | Detail |
|---|---|
| **Purpose** | Present the cultural/mythological story. Clearly label it as NOT science. |
| **What it shows** | Story content with "Cultural Story" label and disclaimer |
| **UI style** | Warm accent color (amber/gold). Scroll/parchment feel but modern. Prominent "Cultural Story" badge at top. Disclaimer visible but not overwhelming. Story text in readable paragraphs. |
| **What makes it impressive** | Feels like opening an ancient story book — but modern and digital. The label is clear so there's no confusion. |
| **Mistakes to avoid** | No label. Label hidden or tiny. Story presented as fact. Too much text without breaks. No visual distinction from science sections. |

---

## Rahasya Chakra (Mystery Wheel)

| Attribute | Detail |
|---|---|
| **Purpose** | Spark wonder and curiosity. Highlight the mystery. |
| **What it shows** | Mystery content + wonder question |
| **UI style** | Deep purple/violet accent. Mysterious feel — subtle glow, question mark icon. Wonder question highlighted or in a callout box. |
| **What makes it impressive** | Makes the user go "hmm, I never thought of that." The wonder question is thought-provoking. Visual style evokes curiosity. |
| **Mistakes to avoid** | Boring content. No wonder question. Same visual style as other cards. No sense of mystery. |

---

## Vigyan Drishti (Science Lens)

| Attribute | Detail |
|---|---|
| **Purpose** | Present modern scientific explanation with verifiable facts. |
| **What it shows** | Science explanation + key facts list |
| **UI style** | Cool blue/cyan accent. Clean, precise, scientific feel. Key facts as distinct bullet points or chips. Icon: telescope or atom. |
| **What makes it impressive** | Clean, authoritative presentation. Key facts are scannable. Feels trustworthy and well-researched. |
| **Mistakes to avoid** | Mixing myth into the science section. No key facts. Wall of text. Informal tone for scientific content. |

---

## Satya Setu (Truth Bridge)

| Attribute | Detail |
|---|---|
| **Purpose** | Show where story and science align, diverge, or remain open. |
| **What it shows** | Bridge content + alignment + divergence |
| **UI style** | Split layout or two-column card. "Alignment" on one side, "Divergence" on the other. Bridge icon or connection visual. Green/teal accent. |
| **What makes it impressive** | Visually shows the connection AND disconnection between story and science. Feels like a balanced, honest assessment. |
| **Mistakes to avoid** | One-sided (only alignment or only divergence). No visual split. Content that dismisses either tradition or science. |

---

## Pramaan Matrix (Evidence Matrix)

| Attribute | Detail |
|---|---|
| **Purpose** | Reality check — what is proven, symbolic, or unknown. |
| **What it shows** | Three categories: Proven, Symbolic, Unknown + evidence level badge |
| **UI style** | Three-column layout or stacked sections. Color-coded: green (proven), yellow (symbolic), gray (unknown). Evidence level as a prominent badge. |
| **What makes it impressive** | Feels like a scientific assessment report. Honest and transparent. The evidence level badge gives a quick visual summary. |
| **Mistakes to avoid** | Always showing "high" evidence. No visual distinction between categories. Missing the "unknown" category. Inflating evidence. |

---

## Drishya Yantra (Visual Instrument)

| Attribute | Detail |
|---|---|
| **Purpose** | Visual representation of the cosmic topic in 3D. |
| **What it shows** | One of 3 3D scenes: cosmic sky, eclipse, planet orbit |
| **UI style** | Embedded 3D canvas, ~300-400px tall, full card width. Dark background. Rounded corners. Subtle animation (rotation, orbit). |
| **What makes it impressive** | Interactive 3D in a web app! Even simple scenes (starfield, orbiting planet) feel impressive. |
| **Mistakes to avoid** | Heavy 3D that crashes mobile. No fallback for non-WebGL browsers. Distracting or too complex. Blocking page load. |

---

## Jigyasa Agni (Curiosity Spark)

| Attribute | Detail |
|---|---|
| **Purpose** | Keep the learner exploring with follow-up questions. |
| **What it shows** | 3 follow-up curiosity questions as clickable cards/buttons |
| **UI style** | Row of 3 cards. Fire/spark themed (orange/red accent). Each question as a clickable card. Hover effect invites clicking. |
| **What makes it impressive** | Feels like a trail of discovery. Each question leads deeper. Creates a "rabbit hole" effect. |
| **Mistakes to avoid** | Questions not clickable. Boring generic questions. Too many questions (keep to 3). No visual distinction. |

---

## Smriti Quest (Memory Quest)

| Attribute | Detail |
|---|---|
| **Purpose** | Test understanding with a quick quiz. |
| **What it shows** | 3–5 multiple choice questions with feedback |
| **UI style** | Card with questions. 4 options per question as buttons. Green for correct, red for incorrect on selection. Explanation revealed after answering. Simple score at the end. |
| **What makes it impressive** | Interactive learning. Immediate feedback. Explanation helps understand mistakes. Feels gamified. |
| **Mistakes to avoid** | No feedback on wrong answers. No explanation. All questions too easy. No visual distinction between correct/incorrect. Quiz blocks further exploration. |
