## Source Verification

All science domain documents are sourced from:
- **NASA** (National Aeronautics and Space Administration) - U.S. government agency, primary authority on space science
- **IAU** (International Astronomical Union) - Global authority on astronomical nomenclature and standards

All narrative domain documents are sourced from:
- **Sacred Texts Archive** (sacred-texts.com) - Established digital library of religious and mythological texts
- **JSTOR** - Academic journal archive with peer-reviewed scholarship on Vedic astronomy and Indian traditions

All boundary and glossary documents are internal policy documents created for Akashgatha's operational guidelines.

## Document Standards

All corpus documents follow strict frontmatter standards validated by `src/lib/server/rag/schema.ts`:

```yaml
---
title: string (required)
domain: "science" | "narrative" | "boundary" | "glossary" (required)
tags: string[] (required, 1-10 tags)
sourceType: "official" | "scholarly" | "traditional" | "internal" (required)
sourceName: string (required)
sourceUrl: string (required for science/narrative, empty string for boundary/glossary)
lastVerified: YYYY-MM-DD (required)
---
```

Validation is enforced via:
```bash
npm run rag:validate
```

## Usage in RAG Pipeline

These documents will be:
1. **Chunked** into semantically coherent segments (Phase 4B-2)
2. **Embedded** using OpenAI text-embedding-3-small model (Phase 4B-3)
3. **Indexed** in Pinecone vector database (Phase 4B-3)
4. **Retrieved** during Jigyasa queries to provide grounded context (Phase 4B-4)

The RAG system ensures that Akashgatha's responses are always grounded in verified sources, maintaining the strict separation between Katha (narrative tradition) and Vigyan (scientific evidence).
