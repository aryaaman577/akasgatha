# Product Requirements Document — AkasGatha

## Product Name

AkasGatha

> **Do not rename.** Do not write "AkasGatha AI." The product name is exactly **AkasGatha**.

## Tagline

*Where ancient sky stories meet modern space science.*

## Vision

AkasGatha is a cinematic educational web platform that brings together ancient Indian sky narratives — planets, nakshatras, eclipses, moon phases, cosmic mysteries — and explains them alongside modern space science. It uses Generative AI to produce structured, layered responses that clearly separate cultural story from scientific evidence.

## Problem Statement

Ancient Indian texts contain rich observations about the sky — planetary movements, eclipses, lunar cycles, constellations. These stories are often presented online either as mythology without scientific context, or misrepresented as literal scientific proof. There is no accessible platform that:

1. Presents these sky stories with cultural respect
2. Explains the modern science behind the same phenomena
3. Clearly separates story, symbolism, science, evidence, and open mystery
4. Uses AI to make this exploration interactive and curiosity-driven

## Target Users

- Students (ages 16–30) interested in space, astronomy, or Indian heritage
- Internship evaluators reviewing Gen AI + Cloud Computing project submissions
- Curious learners who want myth-science bridging without misinformation
- Educators looking for culturally sensitive science communication examples

## Product Goal

Create a **premium but buildable MVP** that:

1. Uses Generative AI (Gemini API) to explain cosmic topics in a structured learning format
2. Deploys on AWS using Docker as proof of cloud computing skills
3. Demonstrates Gen AI prompt engineering, structured output, and content safety
4. Is practical for a student internship final project — not overengineered

## MVP Features

### 1. Akas Dwar (Celestial Gateway)
Cinematic landing section with hero animation, tagline, and call-to-action.

### 2. Akas Granth (Sky Library)
Topic library grid showing available cosmic topics (planets, nakshatras, eclipses, moon phases, mysteries).

### 3. Jigyasa Engine (Curiosity Engine)
Question input form where users type a cosmic question. The AI returns a structured, multi-section answer.

### 4. Katha Mandal (Story Circle)
Response section showing the cultural/mythological story behind the topic. Clearly labeled as "Cultural Story" — not science.

### 5. Rahasya Chakra (Mystery Wheel)
Response section highlighting the mystery, wonder, and curiosity angle of the topic.

### 6. Vigyan Drishti (Science Lens)
Response section presenting the modern scientific explanation with facts and evidence.

### 7. Satya Setu (Truth Bridge)
Response section bridging the story and the science — showing where they align, diverge, or remain open.

### 8. Pramaan Matrix (Evidence Matrix)
Response section providing a reality check — evidence level, what is proven, what is symbolic, what is unknown.

### 9. Drishya Yantra (Visual Instrument)
3 reusable 3D visual scene templates (cosmic sky, eclipse, planet orbit) rendered with React Three Fiber. Mapped to topic type.

### 10. Jigyasa Agni (Curiosity Spark)
Follow-up curiosity questions generated by the AI to keep the learner exploring.

### 11. Smriti Quest (Memory Quest)
Simple quiz/revision section with 3–5 questions per topic to test understanding.

## Out-of-Scope Features (MVP)

These features are explicitly excluded from the MVP:

| Feature | Reason |
|---|---|
| Login / Authentication | Not needed for demo; adds complexity |
| Database | Topics generated on-the-fly via AI; no persistence needed |
| Admin Dashboard | No content management in MVP |
| Music / Voice Narration | Polish feature, not core |
| PDF Generator | Can be manual for report |
| Real Star Map | Requires heavy astronomy APIs |
| Astrology Prediction | Against content safety principles |
| Social Feed | Not a social platform |
| Payment System | Not a commercial product |
| Custom heavy 3D per topic | 3 reusable templates are sufficient |

## User Flow

```
1. User lands on Akas Dwar (hero section)
2. User scrolls to Akas Granth (topic library) or clicks "Ask"
3. User selects a topic OR types a question in Jigyasa Engine
4. Frontend sends POST /api/jigyasa with question + mood + topicType
5. Backend calls Gemini API with structured prompt
6. AI returns structured JSON response
7. Frontend renders response in sections:
   - Katha Mandal (story)
   - Rahasya Chakra (mystery)
   - Vigyan Drishti (science)
   - Satya Setu (bridge)
   - Pramaan Matrix (evidence)
   - Drishya Yantra (3D scene)
   - Jigyasa Agni (follow-up questions)
   - Smriti Quest (quiz)
8. User can ask follow-up questions or explore new topics
```

| Ask / Explore | `/ask` | Jigyasa Engine input + full response view |
| About | `/about` | Project info, content safety note, credits |

## Response Sections (from AI)

Each AI response must return a JSON object with these sections:

```
kathaMandal      — cultural story (clearly labeled)
rahasyaChakra    — mystery/curiosity angle
vigyanDrishti    — modern science explanation
satyaSetu        — bridge between story and science
pramaanMatrix    — evidence check (proven / symbolic / unknown)
sceneType        — recommended 3D scene type for Drishya Yantra
jigyasaAgni      — 3 follow-up curiosity questions
smritiQuest      — 3–5 quiz questions with answers
topicTitle       — display title
topicSummary     — one-line summary
```

## Success Criteria

1. **Functional**: User can ask a cosmic question and receive a structured, multi-section AI response
2. **Content Safety**: No response presents mythology as scientific proof
3. **Visual Quality**: Landing page and response cards look cinematic and premium
4. **3D Scenes**: At least 3 reusable 3D scenes render correctly and map to topic types
5. **Deployment**: App runs in Docker container and deploys to AWS EC2
6. **Report**: Internship report demonstrates Gen AI + Cloud Computing usage
7. **Performance**: Page loads under 3 seconds; 3D scenes load lazily
8. **Security**: API keys not exposed; input validated; rate limiting active

## Non-Negotiable Rules

1. **Product name is AkasGatha.** Never rename. Never write "AkasGatha AI."
2. **Never present mythology as scientific proof.** Every cultural story must be labeled as such.
3. **Never generate astrology predictions.** This is educational, not predictive.
4. **Never expose API keys to the frontend.** All LLM calls happen server-side.
5. **Always validate AI output** against the expected JSON schema before rendering.
6. **Always provide fallback content** if the AI response fails or is malformed.
7. **Separate story from science** in every response — visually and structurally.
8. **Keep it buildable.** Do not add features beyond the MVP scope.
9. **Phase-based development.** Do not skip phases or combine too many at once.
10. **Update brain state files** after completing every phase.
